ПРОСТЫМИ СЛОВАМИ
────────────────
• GitHub — это сайт, куда можно залить папку с проектом (код мода).
• "Репозиторий" — это одна такая папка-проект на GitHub.
• "Ветка" (branch) — как "версия" репозитория. По умолчанию ветка
  называется main или master. Думайте: "главная ветка с кодом".
• "Пуш" (push) — это "отправить свои файлы на GitHub".
  То есть: "загрузить обновлённый код в этот репозиторий".

Когда в workflow написано "при пуше в main/master" — значит:
  "как только ты загрузишь (или обновишь) проект в главную ветку,
   GitHub сам запустит сборку и сделает готовый JAR".


КАК ВСЁ СДЕЛАТЬ ПО ШАГАМ (если никогда не пользовался GitHub)
───────────────────────────────────────────────────────────────

ШАГ 1 — Регистрация
  • Зайди на https://github.com
  • Нажми Sign up и заведи аккаунт (логин, пароль, почта).

ШАГ 2 — Создать репозиторий
  • Войди в аккаунт.
  • Справа вверху нажми зелёную кнопку "New" (или "+" → New repository).
  • Имя репозитория: например  musicplayer-mod  (латиницей).
  • Можешь оставить "Public".
  • НЕ ставь галочку "Add a README" (у тебя уже есть файлы).
  • Нажми "Create repository".

ШАГ 3 — Залить папку с модом
  На GitHub откроется страница с подсказками. Там будет что-то вроде:

    "…or push an existing repository from the command line"

  Под ней команды. Тебе проще всего залить через сайт:

  • Нажми вкладку "uploading an existing file" (или "upload files").
  • Перетащи в окно ВСЮ папку "minecraft mod":
    все файлы и подпапки (build.gradle, src, .github, СБОРКА.txt и т.д.).
  • Внизу в поле описание можно написать: "Music Player mod 1.21.1".
  • Нажми "Commit changes".

  Это и есть твой первый "пуш" в ветку main — ты загрузил код в главную ветку.

ШАГ 4 — Дождаться сборки и скачать JAR
  • В репозитории сверху открой вкладку "Actions".
  • Слева выбери workflow "Build mod".
  • Увидишь один запуск (жёлтый = идёт, зелёная галочка = готово).
  • Кликни по нему.
  • Внизу страницы будет блок "Artifacts".
  • Нажми "musicplayer-mod" — скачается архив. В нём лежит
    musicplayer-1.0.0.jar — его и клади в папку mods.


ЕСЛИ ЗАХОЧЕШЬ ОБНОВИТЬ МОД ПОТОМ
───────────────────────────────────────────────────────────────
  Снова зайди в свой репозиторий → в папку с файлами → "Add file" →
  "Upload files" → перетащи обновлённые файлы → "Commit changes".
  После этого в Actions снова запустится сборка — можно снова скачать
  новый JAR из Artifacts.


КОРОТКО
────────────────
• "Пуш в main/master" = загрузил (или обновил) код в главную ветку на GitHub.
• После этого GitHub сам собирает мод и кладёт готовый JAR в Artifacts.
• Ты просто скачиваешь архив из Actions → Artifacts и достаёшь оттуда JAR.
